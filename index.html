Terraform

A dive into the mind of a person learning how to set up Terraform on GCP for the first time
  1. Create GCP account, utilize 90 days of free tier stuff. I also created a new project called 'Terraform', though feel free to call it whatever you want, or even stick with the original project name given.
  2. Navigate to the 'Compute Engine' for the first time, which will result in loading time. Once complete. create a compute instance under 'Compute Engine'. Using Debian Linux 10 Buster. Picked LA region since it was close enough.
  3. Perform update/upgrade with apt-get
  4. Follow tutorial here: Install Terraform
  5. Run curl -fsSL https://apt.releases.hashicorp.com/gpg | sudo apt-key add -
  6. Attempted sudo apt-add-repository "deb [arch=amd64] https://apt.releases.hashicorp.com $(lsb_release -cs) main", however, apt-add-repository was not a valid command. I then googled 'apt-add-repository', which led me to this page
  7. Ran this command sudo apt-get install software-properties-common
  8. Reran previous apt-add-repository, and now it works fine
  9. Ran the following without any issues: sudo apt-get update && sudo apt-get install terraform
  10. Running terraform -help resulted in the expected help documentation, showing that installation was successful.
  11. Ran terraform -install-autocomplete which helps with typing out commands by pressing tab to autocomplete.
  12. Forgot command on how to restart system. Smh. It's systemctl reboot. However, I now need root password, which I don't know. So as a quick workaround, I went back to the terminal, clicked on the vertical 3 dots, and clicked stop and then start. I almost clicked 'reset', however, I believe this means reset everything back to initial state, which is not what I'm looking for. Should definitely know what my password is though.
  13. Googled 'gcp linux root password' and came to this page
  14. Used sudo passwd and created a new password. Tried systemctl reboot with new password and this time it works. Yay!
  15. Testing autocomplete by typing 'terraform -' and then pressing tab, which then shows all my possible options. Typing 'terraform -h' then tab autocompletes it to 'terraform -help', which means it's working.
  16. As I will not be focusing on Docker at the moment, and the example used in the tutorial is pretty simple, I will be moving on to the next tutorial page
  17. As I already completed the GCP Project and Compute Engine portion already on this page, I will move on with the GCP service account key.
  18. To Be Continued...
